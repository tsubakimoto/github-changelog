## Manage agent skills with GitHub CLI

**URL:** https://github.blog/changelog/2026-04-16-manage-agent-skills-with-github-cli

### 要約
GitHub CLI に `gh skill` が追加され、agent skills を検索・インストール・更新・公開できるようになりました。skills は複数の agent host で使える portable な指示・スクリプト・資源の集合で、GitHub は provenance と pinning も CLI で扱えるようにしています。公開前の機能として、immutability や tree SHA ベースの変更検知も含まれます。

### ユーザーにとって良いこと
AI エージェント向けの作業手順を、リポジトリ横断で再利用しやすくなります。pinning と provenance により、どの skill をどの版で使っているかを追跡しやすくなります。

### アクションプラン
- GitHub CLI を v2.90.0 以降に更新してください。
- `gh skill search` / `gh skill install` で必要な skills を棚卸しし、重要なものは `--pin` で固定してください。
- ⚠️ public preview です。インストール前に skill 内容を確認し、prompt injection や不正なスクリプトがないかレビューしてください。
- skills を公開する側は `gh skill publish --fix` で metadata と設定を整えてください。

### リファレンス

## Rule insights dashboard and unified filter bar

**URL:** https://github.blog/changelog/2026-04-16-rule-insights-dashboard-and-unified-filter-bar

### 要約
repository rulesets 向けに、Settings > Rules で見られる rule insights dashboard が追加されました。成功・失敗・bypass の推移や、最も活発な bypassers を俯瞰でき、各チャートから絞り込み済みの rule insights に直接移動できます。あわせて、複数の alert dismissal / bypass request ページが unified filter bar に統一されました。

### ユーザーにとって良いこと
ルール違反や bypass の傾向を素早く把握でき、インシデント対応や監査がしやすくなります。関連する alert 画面のフィルタ体験も統一され、custom properties を含めて操作しやすくなります。

### アクションプラン
- repository の Settings > Rules を開き、bypass の増加や失敗傾向を定期監視してください。
- code scanning、Dependabot、secret scanning の dismissal / bypass 運用手順を unified filter bar 前提で見直してください。
- ⚠️ public preview です。重要な運用に組み込む前に、対象 repository で挙動を確認してください。

## Claude Opus 4.7 is generally available

**URL:** https://github.blog/changelog/2026-04-16-claude-opus-4-7-is-generally-available

### 要約
Claude Opus 4.7 が GitHub Copilot で generally available になりました。multi-step task、長い推論、tool-dependent な作業でより高い性能を狙ったモデルで、Copilot Pro+、Business、Enterprise で利用できます。今後数週間で Opus 4.5 と 4.6 は model picker から置き換わります。

### ユーザーにとって良いこと
複雑なエージェント作業や長いタスクの成功率向上が期待できます。Copilot CLI や cloud agent を使うワークフローで、より強い reasoning を選べるようになります。

### アクションプラン
- Copilot Enterprise / Business 管理者は Claude Opus 4.7 policy を Copilot settings で有効化してください。
- チームは既存の agent ワークロードで Opus 4.7 を試し、応答品質とコストを比較してください。
- ⚠️ promotional pricing の 7.5× premium request multiplier は 4 月 30 日までです。利用量の多い環境は請求影響を確認してください。
