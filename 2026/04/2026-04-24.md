## Notice about upcoming new format for GitHub App installation tokens

**URL:** https://github.blog/changelog/2026-04-24-notice-about-upcoming-new-format-for-github-app-installation-tokens

### 要約
GitHub App の installation token 形式が、より高性能で安定した新しい stateless 形式に段階的に移行されます。新しいトークンは長さが一定ではなくなり、`ghs_APPID_JWT` 形式になります。

### ユーザーにとって良いこと
高負荷時の発行性能と API の信頼性が改善されます。GitHub 側の内部実装変更により、将来的なスケール改善も受けやすくなります。

### アクションプラン
- `ghs_` トークンを文字列長や正規表現で検証している実装を削除する
- トークン保存用の DB カラムが 520 文字以上を受け入れられるか確認する
- Actions や GitHub App の連携で影響が出た場合に備え、段階的ロールアウト期間中の挙動を監視する
- ⚠️ トークンは不透明な文字列として扱い、内容に依存しない

## Changes to notification retention and archived repository watches

**URL:** https://github.blog/changelog/2026-04-24-changes-to-notification-retention-and-archived-repository-watches

### 要約
Web の通知保持期間が 5 か月から 3 か月に短縮されます。あわせて、6 か月以上アーカイブされたリポジトリで、協力者でないユーザーの watch が解除されます。

### ユーザーにとって良いこと
通知一覧が最近の活動に絞られ、古い通知で埋もれにくくなります。アーカイブ済みリポジトリの watch 管理も整理され、実質的に機能しない watch が減ります。

### アクションプラン
- 通知の長期保管が必要な場合は、Web 以外の保管方法や外部連携を検討する
- アーカイブ済みリポジトリを監視しているチームは、watch 数の減少を想定しておく
- リポジトリを再公開した後に watch が必要なら、再購読手順を案内する
- Email 通知は対象外のため、運用ルールを分けて扱う

### リファレンス
- [Changes to notification retention and archived repository watches](https://github.blog/changelog/2026-04-24-changes-to-notification-retention-and-archived-repository-watches)

## GPT-5.5 is generally available for GitHub Copilot

**URL:** https://github.blog/changelog/2026-04-24-gpt-5-5-is-generally-available-for-github-copilot

### 要約
GitHub Copilot で GPT-5.5 が一般提供されました。複雑なマルチステップの agentic coding タスクに強く、Copilot の複数のクライアントから選択できます。

### ユーザーにとって良いこと
難しい実装や調査タスクで、より高い精度と推論力が期待できます。Copilot CLI を含む複数環境で同じモデルを選べるため、作業体験を揃えやすくなります。

### アクションプラン
- Copilot Business / Enterprise の管理者は GPT-5.5 ポリシーを有効化する
- 利用者はモデルピッカーで GPT-5.5 を選択して評価する
- 7.5× の premium request multiplier を前提に利用量とコストを確認する
- チーム内で、どの作業に GPT-5.5 を使うか運用基準を決める

## Inline agent mode in preview and more in GitHub Copilot for JetBrains IDEs

**URL:** https://github.blog/changelog/2026-04-24-inline-agent-mode-in-preview-and-more-in-github-copilot-for-jetbrains-ides

### 要約
JetBrains IDE 向け Copilot に、Inline agent mode のプレビューや Next Edit Suggestions の改善が追加されました。さらに、auto approve のグローバル設定や、より細かいコマンド・ファイル編集の制御も導入されています。

### ユーザーにとって良いこと
エディタ内でより強力な支援を受けながら、チャットや編集の流れを切らずに作業できます。自動承認の制御が細かくなり、速度と安全性のバランスを取りやすくなります。

### アクションプラン
- JetBrains 利用者は Inline Chat から agent mode を試す
- Business / Enterprise では Editor preview features policy を有効化する
- Auto approve は危険な操作も含むため、既定設定をチームで見直す
- Next Edit Suggestions を使う場合は、IDE 設定で NES を有効化する
