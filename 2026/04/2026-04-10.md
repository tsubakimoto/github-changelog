## Enforcing new limits and retiring Opus 4.6 Fast from Copilot Pro+

**URL:** https://github.blog/changelog/2026-04-10-enforcing-new-limits-and-retiring-opus-4-6-fast-from-copilot-pro

### 要約
Copilot の高負荷利用に対応するため、サービス全体のレート制限と特定モデル/モデルファミリーの利用制限が段階的に導入されます。あわせて、Copilot Pro+ では Opus 4.6 Fast が本日から段階的に終了し、代替として Opus 4.6 の利用が推奨されています。

### ユーザーにとって良いこと
混雑時でも Copilot の応答性と安定性を保ちやすくなります。制限に当たった場合も、代替モデルや Auto mode に切り替えて作業を続けやすくなります。

### アクションプラン
- Copilot を大量利用するチームには、短時間に集中したリクエストを避けるよう案内してください。
- Copilot Pro+ 利用者には、Opus 4.6 Fast の終了と Opus 4.6 への切り替えを周知してください。
- 制限に達したときの運用手順を、待機・再試行・代替モデル利用の観点で更新してください。
- ⚠️ 既存の自動化やバッチ的な利用は、新しい制限で失敗率が上がる可能性があります。

## Copilot usage metrics now aggregate Copilot cloud agent active user counts

**URL:** https://github.blog/changelog/2026-04-10-copilot-usage-metrics-now-aggregate-copilot-cloud-agent-active-user-counts

### 要約
Copilot usage metrics API の enterprise / organization レポートに、Copilot cloud agent のアクティブユーザー集計が追加されました。日次・週次・月次の 3 つの nullable フィールドで、利用状況を期間別に把握できます。

### ユーザーにとって良いこと
ユーザーごとの生データを自前で集計しなくても、cloud agent の利用拡大をすぐに追えます。IDE agent mode や既存の Copilot 指標と並べて、導入状況を立体的に見られます。

### アクションプラン
- usage metrics API を使うダッシュボードに、新しい 3 フィールドを取り込んでください。
- 値が `null` になるケースを前提に、集計ロジックと表示ロジックを確認してください。
- 既存の Copilot 導入レポートに、cloud agent の利用指標を追加してください。
- ⚠️ 既存の月次/週次レポートは、cloud agent 利用を別管理していると数値の見え方が変わります。

## Pausing new GitHub Copilot Pro trials

**URL:** https://github.blog/changelog/2026-04-10-pausing-new-github-copilot-pro-trials

### 要約
不正利用対策のため、新しい GitHub Copilot Pro の無料トライアルが一時停止されます。既存のトライアルは影響を受けず、Copilot Free と有料サブスクリプションは引き続き利用できます。

### ユーザーにとって良いこと
トライアル制度の悪用を抑え、正規の利用者に対するサービス品質を守りやすくなります。継続利用したい人は Copilot Free や有料プランへ明確に誘導されます。

### アクションプラン
- 新規ユーザー向けの案内文や onboarding 手順から、Copilot Pro 試用開始の導線を外してください。
- サポート窓口には、トライアル停止中であることと代替プランを共有してください。
- 既存の販売/案内ページに、Copilot Free と有料プランの案内を明記してください。
- ⚠️ トライアル前提の獲得施策は、停止期間中にそのまま運用できません。

### リファレンス
- [Pausing new GitHub Copilot Pro trials](https://github.blog/changelog/2026-04-10-pausing-new-github-copilot-pro-trials)

## Actions workflows are limited to 50 reruns

**URL:** https://github.blog/changelog/2026-04-10-actions-workflows-are-limited-to-50-reruns

### 要約
GitHub Actions の workflow rerun は 50 回までに制限されます。上限を超えると failed check suite になり、全 rerun と一部 job の rerun の両方が対象です。

### ユーザーにとって良いこと
無限に近い再実行で Actions 基盤に負荷が集中するのを防げます。異常な再試行ループが止まり、全体の安定性が高まります。

### アクションプラン
- rerun を自動化しているワークフローや運用スクリプトがないか確認してください。
- 失敗時は rerun 連打ではなく、原因修正後に再実行する運用へ寄せてください。
- Actions の制限値を監視し、50 回に近づくケースをアラート化してください。
- ⚠️ 既存の再試行フローは、50 回を超えると失敗チェックに変わります。

- [Actions workflows are limited to 50 reruns](https://github.blog/changelog/2026-04-10-actions-workflows-are-limited-to-50-reruns)

## Copilot CLI activity now included in usage metrics totals and feature breakdowns

**URL:** https://github.blog/changelog/2026-04-10-copilot-cli-activity-now-included-in-usage-metrics-totals-and-feature-breakdowns

### 要約
Copilot usage metrics API のトップレベル合計値に、Copilot CLI の利用が統合されました。CLI は `feature=copilot_cli` として feature 系の breakdown にも現れ、IDE のみを表す `totals_by_ide` からは除外されます。

### ユーザーにとって良いこと
CLI 利用も含めた Copilot 全体の実態を、追加の突合なしで把握できます。組織の Copilot 活用度を、IDE と CLI をまたいで一貫した指標で見られるようになります。

### アクションプラン
- Copilot のダッシュボードや定期レポートで、トップレベル総数の意味が変わったことを反映してください。
- CLI を別集計していた場合は、二重計上しないよう集計式を見直してください。
- `totals_by_ide` を IDE 専用指標として使っている箇所は、説明文を更新してください。
- ⚠️ 既存の閾値や前年比較は、CLI 分の追加で数値が上がる可能性があります。

### リファレンス
- [Copilot CLI activity now included in usage metrics totals and feature breakdowns](https://github.blog/changelog/2026-04-10-copilot-cli-activity-now-included-in-usage-metrics-totals-and-feature-breakdowns)

## Copilot cloud agent’s validation tools are now 20% faster

**URL:** https://github.blog/changelog/2026-04-10-copilot-cloud-agents-validation-tools-are-now-20-faster

### 要約
Copilot cloud agent が実行する CodeQL、Advisory Database、secret scanning、Copilot code review の検証が、逐次実行から並列実行に変わりました。検証時間は 20% 短縮され、同じ品質をより速く得られます。

### ユーザーにとって良いこと
cloud agent の完了までの待ち時間が減り、レビュー依頼までの流れが速くなります。検証品質を落とさずに反復回数を増やしやすくなります。

### アクションプラン
- repository settings の Copilot -> Cloud agent で、必要な検証ツールの設定を確認してください。
- cloud agent を使うチームには、検証が速くなったことを周知してください。
- セキュリティ/品質検証の有効化方針がある場合は、並列化後も設定が意図どおりか見直してください。

### リファレンス
- [Copilot cloud agent’s validation tools are now 20% faster](https://github.blog/changelog/2026-04-10-copilot-cloud-agents-validation-tools-are-now-20-faster)
