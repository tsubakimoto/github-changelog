## Prioritize security alerts with runtime context from Dynatrace

**URL:** https://github.blog/changelog/2026-04-07-prioritize-security-alerts-with-runtime-context-from-dynatrace

### 要約
Dynatrace の runtime context を GitHub Advanced Security のアラート優先付けに使えるようになりました。デプロイ済みの成果物や Kubernetes 環境の runtime risk を踏まえて、実際に影響する脆弱性を絞り込めます。

### ユーザーにとって良いこと
「見つかった脆弱性」ではなく「本番で影響する脆弱性」を先に処理しやすくなります。ノイズを減らし、修正の優先順位を運用実態に合わせられます。

### アクションプラン
- Dynatrace を GitHub に接続し、デプロイ情報と runtime risk を取り込んでください。
- code scanning / Dependabot のアラートページや security campaigns で `has:deployment` と `runtime-risk:*` フィルタを使ってください。
- ⚠️ この機能は GitHub Enterprise Cloud 向けです。利用可否を先に確認してください。

## Copilot CLI now supports BYOK and local models

**URL:** https://github.blog/changelog/2026-04-07-copilot-cli-now-supports-byok-and-local-models

### 要約
Copilot CLI が GitHub ホストのモデルルーティングだけでなく、独自のモデルプロバイダーやローカルモデルに対応しました。オフラインモードや GitHub 認証不要の利用も可能になり、エージェント型のターミナル体験をそのまま維持できます。

### ユーザーにとって良いこと
既存のモデル契約をそのまま使えます。エアギャップ環境やローカル実行でも Copilot CLI を使えるため、コスト・コンプライアンス・運用の自由度が上がります。

### アクションプラン
- `copilot help providers` でプロバイダー設定手順を確認し、必要な環境変数を設定してください。
- エアギャップ運用では `COPILOT_OFFLINE=true` を設定し、ローカルモデルと組み合わせてください。
- 使用するモデルが tool calling と streaming をサポートし、可能なら 128k 以上のコンテキストを持つことを確認してください。

## Dependabot alerts are now assignable to AI agents for remediation

**URL:** https://github.blog/changelog/2026-04-07-dependabot-alerts-are-now-assignable-to-ai-agents-for-remediation

### 要約
Dependabot のアラートを Copilot、Claude、Codex などの AI coding agent に割り当てられるようになりました。エージェントは依存関係の影響を調べ、修正案の draft pull request を作成し、テスト失敗の解消も試みます。

### ユーザーにとって良いこと
単純なバージョン更新だけでなく、破壊的変更を伴う修正や複数箇所のコード変更も自動化しやすくなります。脆弱性対応の初動を速めつつ、複雑な修正案を比較できます。

### アクションプラン
- Dependabot alert の詳細画面で **Assign to Agent** を使い、適切な coding agent を選んでください。
- 複数エージェントを同じアラートに割り当て、draft PR の差分を比較してください。
- ⚠️ AI 生成の修正は必ずレビューし、テストと影響範囲を確認してからマージしてください。

### リファレンス
- [Dependabot alerts are now assignable to AI agents for remediation](https://github.blog/changelog/2026-04-07-dependabot-alerts-are-now-assignable-to-ai-agents-for-remediation)

## Dependabot version updates now support the Nix ecosystem

**URL:** https://github.blog/changelog/2026-04-07-dependabot-version-updates-now-support-the-nix-ecosystem

### 要約
Dependabot が Nix flakes の version updates に対応しました。`dependabot.yml` に `nix` を追加すると、`flake.lock` の input を監視して upstream の新しい commit に対する pull request を作成します。

### ユーザーにとって良いこと
Nix の依存更新を手作業で追いかけずに済みます。複数の input を個別 PR で管理できるので、更新の影響を見極めやすくなります。

### アクションプラン
- `.github/dependabot.yml` に `nix` の ecosystem 設定を追加してください。
- `flake.lock` の input が次回の scheduled run で更新されることを確認してください。
- ⚠️ これは version updates であり、security updates ではありません。運用の期待値を分けてください。

## Code scanning: Batch apply security alert suggestions on pull requests

**URL:** https://github.blog/changelog/2026-04-07-code-scanning-batch-apply-security-alert-suggestions-on-pull-requests

### 要約
Pull request の Files changed タブで、code scanning alert の修正提案をまとめて適用できるようになりました。複数の修正を 1 つの commit にまとめることで、個別に scan する回数を減らせます。

### ユーザーにとって良いこと
複数のセキュリティ修正をより速く反映できます。レビューと再スキャンの往復が減るため、PR の流れを止めにくくなります。

### アクションプラン
- 複数の alert を同時に直すときは batch にまとめて適用してください。
- 1 commit にまとめた後の scan 結果を確認し、修正が他の箇所へ影響していないか見てください。
- この機能は github.com で一般提供です。既存の PR 修正フローに組み込んでください。

## npm trusted publishing now supports CircleCI

**URL:** https://github.blog/changelog/2026-04-06-npm-trusted-publishing-now-supports-circleci

### 要約
npm trusted publishing が CircleCI の OIDC provider をサポートしました。これにより、CircleCI からの publish で保存済みの資格情報を使わずに認証できます。あわせて npmjs.com に dark mode も追加されました。

### ユーザーにとって良いこと
長期保存の npm トークンを減らせるため、漏えいリスクを下げられます。CI/CD から直接安全に publish できるので、運用もシンプルになります。

### アクションプラン
- CircleCI で npm を publish している場合は trusted publishing に切り替えてください。
- npm website または `npm trust` CLI で OIDC ベースの設定を行ってください。
- ⚠️ 既存の保存済み credential を使っているワークフローは、移行後に不要な secret を削除してください。

### リファレンス
- [npm trusted publishing now supports CircleCI](https://github.blog/changelog/2026-04-06-npm-trusted-publishing-now-supports-circleci)
