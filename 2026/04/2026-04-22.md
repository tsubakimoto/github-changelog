## Copilot code review user counts now aggregate in usage metrics API

**URL:** https://github.blog/changelog/2026-04-22-copilot-code-review-user-counts-now-aggregate-in-usage-metrics-api

### 要約
Copilot code review の利用状況が、Copilot usage metrics API の enterprise / organization レポートに集計されるようになりました。日次・週次・月次の active / passive ユーザー数を分けて取得でき、手動起点とポリシー自動起点の利用を区別できます。

### ユーザーにとって良いこと
導入の広がりを「自発的な利用」と「ポリシーによる自動利用」に分けて把握できます。これにより、Copilot code review の定着度やポリシーの効果をより正確に追跡できます。

### アクションプラン
- enterprise / organization の usage metrics API レポートに、新しい `*_copilot_code_review_users` フィールドを追加してください。
- 日次・週次・月次で active と passive を分けて、導入効果のダッシュボードを更新してください。
- ⚠️ 既存の集計ロジックが総数前提なら、active 優先のカウント規則に合わせて再計算してください。

## GitHub CLI: Opt-out usage telemetry

**URL:** https://github.blog/changelog/2026-04-22-github-cli-opt-out-usage-telemetry

### 要約
GitHub CLI v2.91.0 から、製品改善のための pseudonymous usage telemetry が送信されるようになりました。送信内容はログモードで確認でき、環境変数または `gh config` でオプトアウトできます。

### ユーザーにとって良いこと
何が送られるかを事前に確認できるため、透明性が高いです。利用状況に応じて CLI の改善が進みやすくなりつつ、必要なら簡単に無効化できます。

### アクションプラン
- `GH_TELEMETRY=log` または `gh config set telemetry log` で送信内容を確認してください。
- 収集を止めたい環境では `GH_TELEMETRY=false`、`DO_NOT_TRACK=true`、または `gh config set telemetry disabled` を設定してください。
- ⚠️ これは GitHub CLI (gh) のみが対象で、Copilot CLI や拡張機能のテレメトリには別設定が必要です。

## Bring your own language model key in VS Code now available

**URL:** https://github.blog/changelog/2026-04-22-bring-your-own-language-model-key-in-vs-code-now-available

### 要約
Copilot Business / Enterprise で、VS Code に BYOK (bring your own language model key) が利用可能になりました。Anthropic、Gemini、OpenAI、OpenRouter、Azure、Ollama、Foundry Local などのモデルを、Chat や plan agent、custom agents から使えます。

### ユーザーにとって良いこと
既存の API キーやローカルモデルをそのまま使えるため、モデル選択の自由度が広がります。Copilot のリクエスト枠を消費せずに、チームの運用ポリシーに合わせた構成を取りやすくなります。

### アクションプラン
- VS Code の Copilot Chat で使うモデル提供元とキー管理方針を決めてください。
- 既定で有効なので、必要なら Copilot policy settings で “Bring Your Own Language Model Key in VS Code” を無効化してください。
- ⚠️ この機能は code completions には適用されません。利用範囲を Chat / agents に限定して案内してください。

### リファレンス
- [Bring your own language model key in VS Code now available](https://github.blog/changelog/2026-04-22-bring-your-own-language-model-key-in-vs-code-now-available)

## C++ code intelligence for GitHub Copilot CLI in public preview

**URL:** https://github.blog/changelog/2026-04-22-c-code-intelligence-for-github-copilot-cli-in-public-preview

### 要約
Copilot CLI で Microsoft C++ Language Server が public preview になりました。C++ のシンボル定義、参照、call hierarchy、型情報を利用でき、grep ベースの探索より正確にコードを理解できます。

### ユーザーにとって良いこと
複雑な include、マクロ、テンプレートを含む C++ コードでも、CLI から関連箇所を見つけやすくなります。IDE に近い精度で探索できるため、大規模 C++ プロジェクトでの調査効率が上がります。

### アクションプラン
- C++ プロジェクトで `compile_commands.json` を用意し、Copilot CLI から参照できるようにしてください。
- npm パッケージの Microsoft C++ Language Server をインストールし、README の手順に従って CLI を設定してください。
- ⚠️ これは public preview です。導入後は検索精度とワークスペース設定を確認し、必要に応じて `Use the C++ LSP` を案内してください。

### リファレンス
- [C++ code intelligence for GitHub Copilot CLI in public preview](https://github.blog/changelog/2026-04-22-c-code-intelligence-for-github-copilot-cli-in-public-preview)
