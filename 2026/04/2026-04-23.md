## Copilot Chat improvements for pull requests

**URL:** https://github.blog/changelog/2026-04-23-copilot-chat-improvements-for-pull-requests

### 要約
Copilot Chat が pull request と diff の文脈をより深く理解できるようになりました。コメント、変更ファイル、commit、review を含む PR の全体像を踏まえて、structured review や要約を返せます。

### ユーザーにとって良いこと
PR の内容確認やレビュー依頼への初動が速くなります。差分の読み込みや論点整理を Copilot に任せやすくなり、レビューの見落としを減らせます。

### アクションプラン
- PR を質問するときは、github.com/copilot か diff 上の Copilot ボタンから使う流れをチームに共有してください。
- レビュー前の確認用に「Help review this pull request」「Summarize this pull request」などの促し文を案内してください。
- ⚠️ diff 上の追加体験は public preview 対象ユーザーのみです。利用可否を環境ごとに確認してください。

## Global pull requests dashboard moves to opt-out public preview

**URL:** https://github.blog/changelog/2026-04-23-global-pull-requests-dashboard-moves-to-opt-out-public-preview

### 要約
グローバルな pull requests ダッシュボードが、既定で有効な opt-out public preview に移行しました。Inbox と一覧表示の両方が改善され、org / repo 単位の絞り込み、下書きやレビュー待ちの専用セクション、更新の可視化などが追加されています。

### ユーザーにとって良いこと
自分の PR を一箇所で追いやすくなり、レビュー依頼や対応待ちの把握がしやすくなります。保存ビューの URL が安定するため、共有やブックマークもしやすいです。

### アクションプラン
- `github.com/pulls` を使うチームは、既定の表示を inbox にするか一覧にするかを見直してください。
- Inbox の org / repo フィルタ、`Your drafts`、`Waiting for review` の使い方をチームに周知してください。
- ⚠️ 既定で有効です。必要なら Preview ラベルまたは feature preview から opt out してください。

## Copilot cloud agent fields added to usage metrics

**URL:** https://github.blog/changelog/2026-04-23-copilot-cloud-agent-fields-added-to-usage-metrics

### 要約
Copilot usage metrics API に `used_copilot_cloud_agent` フィールドが追加されました。既存の `used_copilot_coding_agent` と同じ値を返し、cloud agent 名義に合わせたレポートに移行できます。

### ユーザーにとって良いこと
ダッシュボードや自動集計を、製品名の変更に合わせて整理できます。既存フィールドも残るため、移行を急がずに切り替えられます。

### アクションプラン
- enterprise / organization の usage metrics 取得処理に `used_copilot_cloud_agent` を追加してください。
- 28 日ローリングと 1 日レポートの両方で新旧フィールドの一致を確認してください。
- ⚠️ `used_copilot_coding_agent` は 2026-08-01 まで後方互換のために残ります。移行期限を踏まえて更新してください。

### リファレンス

## Dependabot-based dependency graphs for Python

**URL:** https://github.blog/changelog/2026-04-23-dependabot-graphs-for-python

### 要約
Python プロジェクト向けに、より完全な依存関係ツリーを dependency graph と SBOM に反映できるようになりました。Dependabot の snapshot を Dependency Submission API に送る仕組みで、pip、uv、Poetry をサポートします。

### ユーザーにとって良いこと
間接依存まで含む依存グラフの精度が上がり、可視化や SBOM の信頼性が高まります。actions minutes を消費せずに private registry の設定も活かしやすいです。

### アクションプラン
- Python リポジトリで dependency graph を有効にし、Dependency Submission API の利用を確認してください。
- pip / uv / Poetry のどれを使うかに応じて、Dependabot の設定と private registry の認証を見直してください。
- ⚠️ 依存グラフと SBOM の精度改善が主目的です。既存の依存更新フローを置き換えるものではありません。

### リファレンス
- [Dependabot-based dependency graphs for Python](https://github.blog/changelog/2026-04-23-dependabot-graphs-for-python)

## Disable commit comments across your organization

**URL:** https://github.blog/changelog/2026-04-23-disable-commit-comments-across-your-organization

### 要約
組織レベルで commit comments をまとめて無効化できるようになりました。これまでのリポジトリ単位の設定よりも一括運用しやすくなり、大規模組織での管理コストが下がります。

### ユーザーにとって良いこと
多数の repository を個別に設定変更する手間を減らせます。組織方針として commit comments を禁止したい場合に、設定漏れを避けやすいです。

### アクションプラン
- Organization Settings の Repository > General > Commit comments を確認してください。
- 既に repo ごとに無効化していた場合は、組織設定へ集約して運用を簡素化してください。
- ⚠️ 組織全体に影響します。個別 repo での例外運用が必要かどうかを先に整理してください。

### リファレンス
- [Disable commit comments across your organization](https://github.blog/changelog/2026-04-23-disable-commit-comments-across-your-organization)

## Better debugging with GitHub Copilot on the web

**URL:** https://github.blog/changelog/2026-04-23-better-debugging-with-github-copilot-on-the-web

### 要約
GitHub.com の Copilot Chat で stack trace をより正確に扱えるようになりました。エラーの発生箇所だけでなく、原因、破綻した仮定、最有力の root cause、次の確認点まで整理して返します。

### ユーザーにとって良いこと
障害調査の初動が速くなり、原因切り分けの抜け漏れを減らせます。コードの文脈と合わせて確認できるため、再現しづらい問題でも手がかりを得やすいです。

### アクションプラン
- stack trace を貼る運用を、repo や file context と一緒に案内してください。
- 再現手順や入力値がある場合は、Copilot Chat に合わせて必ず添えてください。
- ⚠️ これは github.com で Copilot を使う全ユーザー向けです。CLI や IDE とは操作場所が異なります。

### リファレンス
- [Better debugging with GitHub Copilot on the web](https://github.blog/changelog/2026-04-23-better-debugging-with-github-copilot-on-the-web)

## View and manage agent sessions from issues and projects

**URL:** https://github.blog/changelog/2026-04-23-view-and-manage-agent-sessions-from-issues-and-projects

### アクションプラン

- Project board の `Show agent sessions` が既定で有効になる点を前提に、表示ルールを見直してください。
- ⚠️ Cloud agent を使っているワークフローに影響します。ログ確認と steer の手順を整えてください。

## Pausing new self-serve signups for GitHub Copilot Business

**URL:** https://github.blog/changelog/2026-04-22-pausing-new-self-serve-signups-for-github-copilot-business

### 要約
GitHub Free と GitHub Team 上の組織向けに、新規の self-serve Copilot Business signup が一時停止されました。既存顧客は影響を受けず、seat の追加と利用は継続できます。

### ユーザーにとって良いこと
既存利用者への影響範囲が明確です。新規導入の可否を事前に判断しやすくなります。

### アクションプラン
- これから Copilot Business を導入する組織は、self-serve 前提で進めないでください。
- 既存顧客は、seat 追加や運用継続に影響がないことを確認してください。
- ⚠️ 新規 self-serve signup は停止中です。再開の案内が出るまで待機してください。

### リファレンス

## GitHub Copilot for Jira: Our latest enhancements

**URL:** https://github.blog/changelog/2026-04-22-github-copilot-for-jira-our-latest-enhancements

### 要約
Jira 連携の Copilot cloud agent が強化されました。カスタム agent、カスタム field、branching rule、space レベルの custom instructions、レビュー通知の Jira コメント通知に対応しています。

### ユーザーにとって良いこと
Jira ticket の文脈をより正確に agent に渡せるため、既存の運用に合わせた自動化がしやすくなります。レビュー待ちも Jira 側で把握しやすくなります。

### アクションプラン
- Jira integration を最新バージョンへ更新してください。
- space レベルの custom instructions で、対象 repo や branch naming rule を明示してください。
- ⚠️ GitHub と Jira の両方で設定が必要です。導入時は通知と branch ルールの整合性を確認してください。
