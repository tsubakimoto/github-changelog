## Organization runner controls for Copilot cloud agent

**URL:** https://github.blog/changelog/2026-04-03-organization-runner-controls-for-copilot-cloud-agent

### 要約
Copilot cloud agent の実行先 runner を、organization 管理者が一元的に制御できるようになりました。デフォルト runner の設定に加えて、repository 側での上書きを禁止するロックも可能です。

### ユーザーにとって良いこと
組織全体で runner の標準化とガードレール設定がしやすくなります。大きい runner での高速化や self-hosted runner への集約を、各 repository ごとの個別設定なしで展開できます。

### アクションプラン
- Organization の Copilot cloud agent 設定で、標準 runner を決めてください。
- 必要なら runner 設定を lock し、repository 側の上書きを禁止してください。
- 自動適用: 既定の runner 選択が organization 単位で反映されます。

## GPT-5.1 Codex, GPT-5.1-Codex-Max, and GPT-5.1-Codex-Mini deprecated

**URL:** https://github.blog/changelog/2026-04-03-gpt-5-1-codex-gpt-5-1-codex-max-and-gpt-5-1-codex-mini-deprecated

### 要約
Copilot 全体で GPT-5.1 系 Codex モデルが非推奨になりました。代替は GPT-5.3-Codex に案内されており、Chat、inline edits、ask/agent モード、code completions に影響します。

### ユーザーにとって良いこと
サポート対象のモデルへ切り替えることで、Copilot の機能を継続利用できます。Enterprise 管理者は、モデルポリシーを見直すことで利用可否を制御できます。

### アクションプラン
- ワークフローや統合で GPT-5.1 系を指定している箇所を GPT-5.3-Codex に更新してください。
- Copilot Enterprise 管理者は、モデルポリシーで代替モデルの利用を有効化してください。
- ⚠️ 既に 2026-04-01 に deprecated されているため、早急に置き換えてください。

## Organization firewall settings for Copilot cloud agent

**URL:** https://github.blog/changelog/2026-04-03-organization-firewall-settings-for-copilot-cloud-agent

### 要約
Copilot cloud agent の firewall 設定を organization 管理者が一括で管理できるようになりました。firewall の ON/OFF、recommended allowlist、custom allowlist、repository 管理者の追加権限まで制御できます。

### ユーザーにとって良いこと
内部リソースへのアクセスを含め、組織のセキュリティ方針を全 repository に反映しやすくなります。既定値は従来どおり repository ごとに選べるため、既存運用を壊しにくいです。

### アクションプラン
- Organization 全体の firewall 既定値を決めてください。
- 内部パッケージレジストリなどが必要なら、organization-wide allowlist を追加してください。
- Repository 管理者に custom allowlist 追加を許可するかどうかを明確にしてください。
- 自動適用: 既定は「各 repository が決める」になっています。

## Copilot cloud agent signs its commits

**URL:** https://github.blog/changelog/2026-04-03-copilot-cloud-agent-signs-its-commits

### 要約
Copilot cloud agent が作成する commit に署名するようになりました。署名済み commit は Verified として表示され、Require signed commits を有効にした branch protection や ruleset でも利用できます。

### ユーザーにとって良いこと
agent が作った commit の真正性を確認しやすくなります。これまで使えなかった signed commits 必須の repository でも、Copilot cloud agent を利用できるようになります。

### アクションプラン
- Require signed commits を使っている repository で、Copilot cloud agent を再利用できるか確認してください。
- Verified 表示が期待どおり出るか、重要な workflow で検証してください。
- 自動適用: Copilot cloud agent の commit が署名付きになります。

- [Copilot cloud agent signs its commits](https://github.blog/changelog/2026-04-03-copilot-cloud-agent-signs-its-commits)
