## CodeQL now supports sanitizers and validators in models-as-data

**URL:** https://github.blog/changelog/2026-04-21-codeql-now-supports-sanitizers-and-validators-in-models-as-data

### 要約
CodeQL の models-as-data で、サニタイザとバリデータを宣言的に定義できるようになりました。`barrierModel` と `barrierGuardModel` を使うことで、C/C++、C#、Go、Java/Kotlin、JavaScript/TypeScript、Python、Ruby、Rust の taint 解析に独自の安全判定を組み込めます。

### ユーザーにとって良いこと
自分たちのコードベースにあるエスケープ処理や入力検証を CodeQL に認識させやすくなり、コードスキャンの精度が上がります。カスタム CodeQL コードを書かずに済むため、保守負担を抑えながら誤検知の削減や検出範囲の拡張がしやすくなります。

### アクションプラン
- CodeQL モデルパック内で、既存のサニタイズ関数や検証関数を `barrierModel` / `barrierGuardModel` で定義してください。
- 既存のカスタム CodeQL 実装で barrier を書いている場合は、YAML の data extension へ移行してください。
- ⚠️ 影響は CodeQL モデルパックを使っているリポジトリに限定されます。導入後は code scanning の結果を確認し、期待どおりに taint が遮断されるか検証してください。
