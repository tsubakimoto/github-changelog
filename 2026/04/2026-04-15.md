## CodeQL 2.25.2 adds Kotlin 2.3.20 support and other updates

**URL:** https://github.blog/changelog/2026-04-15-codeql-2-25-2-adds-kotlin-2-3-20-support-and-other-updates

### 要約
CodeQL 2.25.2 が公開され、Kotlin 2.3.20 の分析サポートが追加されました。あわせて、複数言語で false positive を減らすクエリ改善と、@security-severity の見直しが行われています。GitHub code scanning に利用される CodeQL の更新は、github.com 利用者へ自動展開されます。

### ユーザーにとって良いこと
誤検知が減ることで、修正が必要な alert に集中しやすくなります。Kotlin の新しいバージョンに対応したことで、より新しいコードベースでも分析を継続しやすくなります。

### アクションプラン
- code scanning を使っているリポジトリで、Kotlin 2.3.20 以降のコードが正しく分析されるか確認してください。
- 既存の alert の重大度が変わる可能性があるため、triage ルールと優先順位を見直してください。
- ⚠️ この更新は github.com の code scanning 利用者に自動適用されます。GHES の古い環境では手動アップグレードを検討してください。

## Enable Copilot cloud agent via custom properties

**URL:** https://github.blog/changelog/2026-04-15-enable-copilot-cloud-agent-via-custom-properties

### 要約
Copilot cloud agent を、enterprise 全体の一括設定だけでなく、organization 単位で選択的に有効化できるようになりました。organization custom properties を使った条件付き有効化にも対応し、新しい REST API と AI Controls 画面から管理できます。なお、custom properties による判定は設定時点の一回限りで評価されます。

### ユーザーにとって良いこと
一部の organization だけで段階的に導入できるため、リスクを抑えながら展開できます。チームごとに試験導入し、その後に対象を広げる運用がしやすくなります。

### アクションプラン
- enterprise 管理者は、AI Controls で `Enabled for selected organizations` のポリシーを確認してください。
- custom properties を使って対象 organization を絞る場合は、設定後に自動追従しない点を前提に運用してください。
- API 連携を使う場合は、新しい PUT / POST / DELETE エンドポイントに合わせて管理スクリプトを更新してください。
