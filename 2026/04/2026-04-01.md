## Research, plan, and code with Copilot cloud agent

**URL:** https://github.blog/changelog/2026-04-01-research-plan-and-code-with-copilot-cloud-agent

### 要約
Copilot cloud agent が、プルリクエストを作らずにブランチ上で作業できるようになりました。あわせて、実装前に計画だけを作らせてレビューする流れや、コードベースを深く調査する research セッションも使えます。PR は必要になった時点で作成でき、最初から PR を作る指定も可能です。

### ユーザーにとって良いこと
実装前に方針を確認できるため、手戻りを減らしやすくなります。PR 起点に縛られないので、探索・試作・レビューの順序を柔軟に組めます。

### アクションプラン
- Copilot の agent ワークフローを、PR 前提ではなく「ブランチ作業 → Diff確認 → 必要時にPR作成」に切り替えてください。
- 重要な変更では、まず plan を生成させてから実装を進めてください。
- Business / Enterprise 利用者は、管理者が Copilot cloud agent を有効化しているか確認してください。
- オプトイン: プロンプトで PR 作成を明示すると、最初から PR を作れます。

- [Research, plan, and code with Copilot cloud agent](https://github.blog/changelog/2026-04-01-research-plan-and-code-with-copilot-cloud-agent)

## Codespaces is now generally available for GitHub Enterprise with data residency

**URL:** https://github.blog/changelog/2026-04-01-codespaces-is-now-generally-available-for-github-enterprise-with-data-residency

### 要約
GitHub Enterprise Cloud with data residency 向けの Codespaces が一般提供になりました。対応リージョンは Australia、EU、Japan、US で、通常の GitHub platform 上の Codespaces と同等の機能を利用できます。データレジデンシーを厳格に保つため、enterprise か organization 所有の codespaces が必須です。

### ユーザーにとって良いこと
コンプライアンス要件を満たしながら、すぐに使えるクラウド開発環境を導入しやすくなります。リージョン制約がある組織でも、標準機能に近い形で開発体験を揃えられます。

### アクションプラン
- data residency を使う組織は、Codespaces の所有者を enterprise か organization に設定してください。
- user-owned codespaces を前提にした運用がないか確認してください。これはサポートされません。
- Codespaces の所有権ポリシーを、組織のコンプライアンス設定に合わせて見直してください。
- 自動適用: 対応リージョンでは機能が利用可能になりますが、所有権の設定は必要です。

## GPT-5.4 mini is now available in Copilot Student auto model selection

**URL:** https://github.blog/changelog/2026-04-01-gpt-5-4-mini-is-now-available-in-copilot-student-auto-model-selection

### 要約
Copilot Student plan の Auto model selection に GPT-5.4 mini が追加されました。VS Code、Visual Studio、JetBrains、Xcode、Eclipse の Copilot Chat で Auto 経由で利用できます。モデル選択を手動で変えなくても、Student 向けの自動選択に含まれます。

### ユーザーにとって良いこと
Student 利用者は、より新しいモデルを追加設定なしで使える可能性が高まります。IDE ごとに同じ Auto 体験を維持しながら、Copilot の応答品質向上を受けられます。

### アクションプラン
- Copilot Student plan 利用者は、Copilot Chat の model selector で Auto を使ってください。
- IDE 側で Copilot Chat を使っている場合は、対応クライアントが最新か確認してください。
- 管理者対応は通常不要ですが、利用可能モデルの案内文は更新してください。
- 自動適用: Student の Auto 選択に順次反映されます。

### リファレンス
- [GPT-5.4 mini is now available in Copilot Student auto model selection](https://github.blog/changelog/2026-04-01-gpt-5-4-mini-is-now-available-in-copilot-student-auto-model-selection)

## GitHub Mobile: Stay in flow with a refreshed Copilot tab and native session logs

**URL:** https://github.blog/changelog/2026-04-01-github-mobile-stay-in-flow-with-a-refreshed-copilot-tab-and-native-session-logs

### 要約
GitHub Mobile の Copilot 体験が刷新され、ナビゲーションバーから Copilot タブに入りやすくなりました。セッション一覧、フィルタ、ネイティブの session logs、完了セッションからの PR 作成、実行中セッションの停止などがアプリ内で行えます。iOS / Android の最新 production build で利用できます。

### ユーザーにとって良いこと
外出先でも、agent セッションの進捗確認と次の操作をアプリ内で完結しやすくなります。ログ確認や PR 化のために Web に戻る手間が減ります。

### アクションプラン
- GitHub Mobile を最新の production build に更新してください。
- モバイルで agent を運用しているチームは、session logs と state フィルタを使う運用に切り替えてください。
- PR 作成やセッション停止をモバイルで行う担当者に、新しい導線を案内してください。
- 自動適用: 最新版アプリで機能が有効になります。

## GitHub Mobile: Faster, more flexible agent assignment from issues

**URL:** https://github.blog/changelog/2026-04-01-github-mobile-faster-more-flexible-agent-assignment-from-issues

### 要約
GitHub Mobile の issue メニューから、より簡単に agent を割り当てられるようになりました。割り当て時にカスタム指示や別リポジトリ指定ができ、新規 issue 作成時にも同じオプションが使えます。issue 作成と引き継ぎの流れがモバイルで完結しやすくなりました。

### ユーザーにとって良いこと
issue からの作業依頼を、移動中でもすばやく agent に渡せます。指示内容や対象リポジトリをその場で調整できるため、依頼の精度が上がります。

### アクションプラン
- issue をモバイルで起票する担当者に、Assign an Agent の新オプションを案内してください。
- カスタム指示と別リポジトリ指定の運用ルールを整備してください。
- 既存の issue triage 手順に、モバイルでの agent 割り当てを追加してください。
- 自動適用: 最新版 GitHub Mobile で利用できます。

### リファレンス

## Upcoming deprecation of Claude Sonnet 4 in GitHub Copilot

**URL:** https://github.blog/changelog/2026-03-31-upcoming-deprecation-of-claude-sonnet-4-in-github-copilot

### 要約
Claude Sonnet 4 が、2026-05-01 に GitHub Copilot 全体で非推奨になります。Copilot Chat、inline edits、ask/agent modes、code completions のすべてが対象で、代替は Claude Sonnet 4.6 です。Copilot Enterprise 管理者は、代替モデルを model policy で有効化する必要がある場合があります。

### ユーザーにとって良いこと
早めに代替モデルへ移行すれば、停止日以降の利用中断を避けられます。モデルを統一しておくことで、チーム内の Copilot 体験も安定します。

### アクションプラン
- ⚠️ 2026-05-01 までに、Claude Sonnet 4 を使う workflow と integration を Claude Sonnet 4.6 に切り替えてください。
- Copilot Enterprise 管理者は、必要な代替モデルが model policy で有効か確認してください。
- VS Code と github.com の Copilot Chat で、対象モデルが選べるか事前に確認してください。
- 非推奨: 期限後は Claude Sonnet 4 の利用継続を前提にしないでください。

### リファレンス
- [Upcoming deprecation of Claude Sonnet 4 in GitHub Copilot](https://github.blog/changelog/2026-03-31-upcoming-deprecation-of-claude-sonnet-4-in-github-copilot)
