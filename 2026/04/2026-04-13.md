## Copilot data residency in US + EU and FedRAMP compliance now available

**URL:** https://github.blog/changelog/2026-04-13-copilot-data-residency-in-us-eu-and-fedramp-compliance-now-available

### 要約
Copilot のデータ居住性が US と EU に対応し、指定した地域内で inference 処理と関連データを保持できるようになりました。あわせて、US government 向けには FedRAMP Moderate 準拠のモデルホストとインフラが提供されます。主要な Copilot 機能は対象で、地域別の利用可能モデルも用意されています。

### ユーザーにとって良いこと
規制要件やデータ所在地の制約がある組織でも、Copilot を導入しやすくなります。コンプライアンスを満たしながら、Chat や agent mode、code review などの主要機能を使えるのが大きな利点です。

### アクションプラン
- Enterprise / organization admin は、Copilot settings で data residency と FedRAMP policies を明示的に有効化してください。
- 利用予定のモデルが US / EU のデータ居住性対応か確認し、Gemini は現時点で未対応である点を周知してください。
- ⚠️ ポリシーは既定でオフのため、コンプライアンス前提の組織は手動設定が必要です。
- ⚠️ data residency と FedRAMP のリクエストは model multiplier が 10% 増えるため、予算見積もりを更新してください。

## Fix merge conflicts in three clicks with Copilot cloud agent

**URL:** https://github.blog/changelog/2026-04-13-fix-merge-conflicts-in-three-clicks-with-copilot-cloud-agent

### 要約
github.com から新しい Fix with Copilot ボタンを使って、merge conflicts の解消を Copilot cloud agent に依頼できるようになりました。コメントを送るだけで、Copilot が競合を修正し、ビルドとテストを確認して、変更を push します。PR 上では failing workflow の修正やレビューコメント対応など、追加の変更依頼にも使えます。

### ユーザーにとって良いこと
競合解消や軽微な修正にかかる手作業が減り、PR を前に進める速度が上がります。クラウド環境で検証までまとめて処理されるため、ローカルの準備なしに反復しやすくなります。

### アクションプラン
- Copilot Business / Enterprise 利用者向けには、管理者が Copilot cloud agent を有効化しているか確認してください。
- PR 運用手順に、merge conflict 時の Fix with Copilot と `@copilot` の使い方を追加してください。
- 自動修正後も、必要に応じて差分と結果をレビューする流れを残してください。
- ⚠️ この機能は paid Copilot plans で利用でき、Business / Enterprise では管理者設定が必要です。

## Remote control CLI sessions on web and mobile in public preview

**URL:** https://github.blog/changelog/2026-04-13-remote-control-cli-sessions-on-web-and-mobile-in-public-preview

### 要約
`copilot --remote` により、実行中の Copilot CLI セッションを GitHub の web や Mobile から監視・操作できるようになりました。セッションの activity をリアルタイムで確認しながら、追加のコマンド送信、plan の確認、モード切り替え、権限承認などが行えます。操作内容は CLI と GitHub の間で同期されます。

### ユーザーにとって良いこと
長時間タスクを別端末から追えるので、移動中や離席中でも作業を止めにくくなります。セッションの steering や権限応答を外出先から扱えるため、Copilot CLI の使い勝手が大きく広がります。

### アクションプラン
- `/update` で Copilot CLI を最新版に更新してください。
- 新しい remote session は `copilot --remote` で開始し、既存セッションでは `/remote` を使って有効化してください。
- 長時間実行する作業では `/keep-alive` を使い、端末のスリープを防いでください。
- ⚠️ Business / Enterprise 利用者は、remote control と CLI policies を管理者が有効化している必要があります。
