## Copilot SDK in public preview

**URL:** https://github.blog/changelog/2026-04-02-copilot-sdk-in-public-preview

### 要約
GitHub Copilot SDK が public preview になりました。Copilot の agentic な機能を、自分のアプリやワークフロー、プラットフォームサービスに組み込めます。tool 呼び出し、ストリーミング、ファイル操作、マルチターンセッションなどを、Copilot cloud agent や Copilot CLI と同系統の実行基盤で使えます。

### ユーザーにとって良いこと
AI エージェント基盤を一から作らずに、Copilot の実行能力を既存の製品や内部ツールへ持ち込めます。言語別 SDK がそろっているため、実装言語に合わせて導入しやすくなります。

### アクションプラン
- Node.js / TypeScript、Python、Go、.NET、Java のいずれかで SDK を選び、`@github/copilot-sdk` などのパッケージを導入してください。
- ドメイン固有の tool と agent を定義し、必要な場面でだけ呼ばせる設計にしてください。
- ⚠️ public preview なので、本番投入前に approval handler や permissions 設計を含めて検証してください。

## Copilot usage metrics now includes per-user GitHub Copilot CLI activity in organization reports

**URL:** https://github.blog/changelog/2026-04-02-copilot-usage-metrics-now-includes-per-user-github-copilot-cli-activity-in-organization-reports

### 要約
Organization レポートで、ユーザーごとの GitHub Copilot CLI 利用状況を確認できるようになりました。1 日・28 日レポートの両方で、CLI の有無、セッション数、リクエスト数、トークン使用量、平均トークン数、最新 CLI バージョンが見られます。

- CLI 利用者の分布と version のばらつきを確認し、アップグレード対象を絞ってください。
- 自動適用: 組織レポートに新しい内訳が追加されます。

## GitHub Actions: Early April 2026 updates

**URL:** https://github.blog/changelog/2026-04-02-github-actions-early-april-2026-updates

### 要約
GitHub Actions に、service container の entrypoint / command 上書きが追加されました。あわせて、OIDC トークンの repository custom properties が一般提供になり、Azure private networking の VNET failover は public preview になっています。

### ユーザーにとって良いこと
service container の挙動をワークフローから直接制御しやすくなり、回避策が減ります。OIDC と Azure private networking の拡張で、クラウドアクセス制御と可用性も強化されます。

### アクションプラン
- service container を使う workflow は、必要に応じて `entrypoint` と `command` を `yaml` で明示してください。
- cloud provider の OIDC trust policy を、repository custom properties を使う形に見直してください。
- Azure private networking を使っている場合は、secondary subnet と failover 手順を用意してください。
- ⚠️ VNET failover は public preview なので、手動切り戻しの運用責任も含めて確認してください。

### リファレンス
- [GitHub Actions: Early April 2026 updates](https://github.blog/changelog/2026-04-02-github-actions-early-april-2026-updates)

## The Security tab is now Security & quality

**URL:** https://github.blog/changelog/2026-04-02-the-security-tab-is-now-security-quality

### 要約
github.com の Security タブが Security & quality に名称変更されました。Vulnerability alerts は Findings になり、Code quality セクションと Security policy への名称変更も行われています。URL や API はそのままで、GitHub Enterprise Server には含まれません。

### ユーザーにとって良いこと
セキュリティと品質の問題を同じ場所で見つけやすくなります。既存のリンクや自動化を壊さずに、ナビゲーションだけが整理されます。

### アクションプラン
- UI 文言や社内ドキュメントの「Security tab」表記を「Security & quality」に更新してください。
- repository sidebar の「Vulnerability alerts」前提で作った説明や手順があれば、Findings に合わせて直してください。
- GHES 利用者は、この変更が github.com 限定である点を周知してください。
- 自動適用: URL と API は変更されません。

## Improved search for GitHub Issues is now generally available

**URL:** https://github.blog/changelog/2026-04-02-improved-search-for-github-issues-is-now-generally-available

### 要約
GitHub Issues の improved search が一般提供になりました。issue の title と body を意味ベースで検索でき、dashboard と repository 内の両方で使えます。REST と GraphQL API でも semantic / hybrid 検索が利用できます。

### ユーザーにとって良いこと
キーワードが完全一致しなくても、探したい issue を見つけやすくなります。検索結果の関連性が上がり、issue triage や自動化に組み込みやすくなります。

### アクションプラン
- issue 管理ツールやスクリプトで、必要に応じて `search_type=semantic` または `searchType: SEMANTIC` を使ってください。
- 精密検索が必要な場面では、filters や quotation marks を使う lexical search に切り替えてください。
- semantic / hybrid 検索は 10 requests/min の制限があるため、バッチ実行の間隔を調整してください。
- 自動適用: Issues dashboard と API で利用可能になります。

### アクションプラン
- Organization の Copilot 設定で custom instructions を作成し、標準ルールを明文化してください。
- Copilot Chat、code review、cloud agent の出力にどう効くかをチームで確認してください。
- Business / Enterprise の管理者は、既存の運用ガイドをこの機能に合わせて更新してください。
- 自動適用: 対象 organization で利用可能になります。
