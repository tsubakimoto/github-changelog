## Copilot-reviewed pull request merge metrics now in the usage metrics API

**URL:** https://github.blog/changelog/2026-04-08-copilot-reviewed-pull-request-merge-metrics-now-in-the-usage-metrics-api

### 要約
Copilot usage metrics API に、Copilot code review が関与した pull request の新しい指標が追加されました。具体的には、Copilot によってレビューされてマージされた PR の総数と、その PR の作成からマージまでの中央値時間を取得できます。

### ユーザーにとって良いこと
Copilot のレビューがマージ速度やレビュー採用率にどう影響しているかを、既存の作成系メトリクスと並べて確認できます。レビューの価値を定量化しやすくなります。

### アクションプラン
- Copilot usage metrics を利用している場合は、新しい `pull_requests.total_merged_reviewed_by_copilot` と `pull_requests.median_minutes_to_merge_copilot_reviewed` をレポートに追加してください。
- 既存の Copilot 作成 PR 指標と比較し、レビューと作成の両面での効果を確認してください。
- これらの指標は enterprise / organization レベルの 1 日および 28 日ロールिंगウィンドウで利用できます。

## Secret scanning improvements to alert APIs, webhooks, and delegated workflows

**URL:** https://github.blog/changelog/2026-04-08-secret-scanning-improvements-to-alert-apis-webhooks-and-delegated-workflows

### 要約
Secret scanning の alert API、webhook、委任ワークフローが拡張されました。`exclude_secret_types` フィルタ、location の `html_url`、委任クローズ要求のコメント関連フィールド、メール通知の改善、`resolution_comment` の修正が含まれます。

### ユーザーにとって良いこと
自動化や通知がより扱いやすくなり、API 呼び出しや webhook 受信側で追加の取得処理が減ります。委任フローの可視性も上がります。

### アクションプラン
- secret scanning の取得処理がある場合は、`exclude_secret_types` を使って除外ベースの絞り込みに切り替えてください。
- webhook / API の location 処理で `html_url` を優先し、追加のリンク組み立てを減らしてください。
- delegated closure を扱う実装では `closure_request_comment`、`closure_request_reviewer_comment`、`closure_request_reviewer` を受け取れるように更新してください。
- `resolution_comment` に依存する監査・同期処理は、今回の修正後の挙動を前提にしてください。

## GitHub Mobile: Research and code with Copilot cloud agent anywhere

**URL:** https://github.blog/changelog/2026-04-08-github-mobile-research-and-code-with-copilot-cloud-agent-anywhere

### 要約
GitHub Mobile で Copilot cloud agent が pull request 作成だけでなく、コードベースの調査、実装計画の作成、ブランチ上での変更までできるようになりました。必要なら、セッション完了時に PR を自動作成できます。

### ユーザーにとって良いこと
デスクにいないときでも、調査から実装、レビュー、PR 作成までをモバイルで進めやすくなります。作業の中断が減り、移動中でも開発を前進できます。

### アクションプラン
- GitHub Mobile を最新に更新してください。
- モバイルで Copilot に「調査」「計画」「ブランチ編集」「PR 自動作成」を指示する運用を試してください。
- PR を即作成したくない場合は、プロンプトでその旨を明示してください。

- [GitHub Mobile: Research and code with Copilot cloud agent anywhere](https://github.blog/changelog/2026-04-08-github-mobile-research-and-code-with-copilot-cloud-agent-anywhere)

## Code Security risk assessment available for organizations

**URL:** https://github.blog/changelog/2026-04-08-code-security-risk-assessment-available-for-organizations

### 要約
Organization 向けに無料の Code Security risk assessment が提供されました。脆弱性を severity、rule type、language 別に整理し、Copilot Autofix が役立つ箇所も示します。

### ユーザーにとって良いこと
セキュリティ上の優先順位付けがしやすくなり、影響の大きいリポジトリから効率よく対処できます。修正候補の把握にも役立ちます。

### アクションプラン
- Organization の Security タブ内の「Assessments」から評価を開始してください。
- 結果を見て、重大度の高い問題と高影響リポジトリを優先してください。
- Copilot Autofix の適用候補があれば、修正提案の活用を検討してください。

## New PGP signing key for GitHub CLI Linux packages

**URL:** https://github.blog/changelog/2026-04-08-new-pgp-signing-key-for-github-cli-linux-packages

### 要約
GitHub CLI の Linux パッケージ向けに、既存の署名鍵と新しい置き換え鍵を含む更新済み PGP keyring が公開されました。apt、yum、dnf で gh を入れている場合は、期限前に再インストール手順を実行する必要があります。

### ユーザーにとって良いこと
Linux パッケージの署名検証を継続でき、将来の更新停止を避けられます。Windows/macOS や Homebrew、Conda、バイナリ利用者には影響しません。

### アクションプラン
- apt / yum / dnf で gh を導入している場合は、2026-09-05 までに配布元の再インストール手順を実行してください。
- インストール手順を自動化している場合は、新しい keyring を取り込むよう更新してください。
- すでに新しい keyring 公開後に再インストール済みなら追加対応は不要です。

- [New PGP signing key for GitHub CLI Linux packages](https://github.blog/changelog/2026-04-08-new-pgp-signing-key-for-github-cli-linux-packages)

## GitHub Copilot in Visual Studio Code, March Releases

**URL:** https://github.blog/changelog/2026-04-08-github-copilot-in-visual-studio-code-march-releases

### 要約
VS Code の Copilot 関連更新がまとめて案内されました。Autopilot、組み込みブラウザでのデバッグ、画像・動画対応、統合カスタマイズエディタ、agent permissions、nested subagents などが含まれます。

### ユーザーにとって良いこと
エージェント作業の自律性やカスタマイズ性が上がり、ローカル開発や Copilot CLI との連携も広がります。レビューやデバッグの流れも VS Code 内で完結しやすくなります。

### アクションプラン
- VS Code を更新して、新しい Copilot 機能を確認してください。
- 自律実行を試す場合は、セッションごとの permission level と Autopilot の使い方をチームで揃えてください。
- MCP サーバーや custom agents / skills を使っている場合は、統合エディタとモノレポ検出の挙動を確認してください。
- Copilot CLI を併用しているなら、VS Code 側の MCP 設定がそのまま使える前提で運用を見直してください。
