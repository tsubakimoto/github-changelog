## Changes to GitHub Copilot plans for individuals

**URL:** https://github.blog/changelog/2026-04-20-changes-to-github-copilot-plans-for-individuals

### 要約
GitHub Copilot の個人向けプランに変更が入り、新規の Pro / Pro+ / Student 申し込みが一時停止されます。あわせて個人向けプランの利用上限が引き締められ、Pro では Opus モデルが利用できなくなります。

### ユーザーにとって良いこと
既存の有料ユーザーに対して、より安定したサービス提供を優先する方針が明確になりました。上限超過が近づくと VS Code や Copilot CLI で警告が出るため、必要に応じて早めにプランや利用方法を見直せます。

### アクションプラン
- Copilot Pro / Pro+ を使っているチームは、現在の利用量が新しい上限に収まるか確認してください。
- 高い利用上限や Opus モデルが必要な場合は、Pro+ への移行を検討してください。
- ⚠️ 既存の運用が Opus モデル前提なら、代替モデルへの切り替え計画を用意してください。

## Sunsetting SHA-1 in HTTPS on GitHub

**URL:** https://github.blog/changelog/2026-04-20-sunsetting-sha-1-in-https-on-github

### 要約
GitHub は HTTPS 通信での SHA-1 利用を段階的に終了します。2026-07-14 に brownout があり、2026-09-15 には github.com とパートナー CDN で完全に無効化されます。

### ユーザーにとって良いこと
より現代的で安全な TLS 構成へ移行することで、GitHub への接続の信頼性と将来互換性が高まります。影響を受ける環境は事前に把握でき、期限までに更新作業を進められます。

### アクションプラン
- ブラウザ、Git クライアント、API クライアントを最新の TLS 対応版に更新してください。
- `https://github.dev` にアクセスして、現在の環境が SHA-1 非依存の HTTPS に対応しているか確認してください。
- ⚠️ 2026-07-14 の brownout までに、古い TLS 実装を使う社内ツールや自動化を洗い出して修正してください。
