## Figma MCP server can now generate design layers from VS Code

**URL:** https://github.blog/changelog/2026-03-06-figma-mcp-server-can-now-generate-design-layers-from-vs-code

### 要約
GitHub Copilot ユーザーが Figma MCP サーバーに接続することで、デザインコンテキストをコードに取り込むだけでなく、レンダリングされた UI を編集可能なフレームとして Figma に送信できるようになりました。これにより、デザインからコードを生成し、UI を Figma に送り返して反復し、更新をコードベースに取り込むという連携ワークフローが実現します。VS Code で本日利用可能、Copilot CLI でも近日提供予定です。

### ユーザーにとって良いこと
- デザインとコードの往復が一つのツールチェインで完結し、デザイナーと開発者のコラボレーションが大幅に効率化されます
- Figma のデザインをコードに変換するだけでなく、実装した UI を編集可能な形式で Figma に戻せるため、デザインレビューと反復が高速化されます
- すべての Figma プランで利用可能なため、追加コストなしでこの機能を利用できます

### アクションプラン
- **Figma MCP サーバーをインストール**: [公式ドキュメント](https://github.blog/changelog/2026-03-06-figma-mcp-server-can-now-generate-design-layers-from-vs-code)を参照して Figma MCP サーバーをセットアップ
- **Figma アカウントを接続**: VS Code で Figma アカウントを認証し、MCP サーバーとの連携を有効化
- **Copilot でデザインを取り込む**: Copilot を使って Figma のデザインをコードに変換するプロンプトを試す
- **UI を Figma に送信**: レンダリングされた UI を Figma に送り返す機能を試し、デザイナーとの反復プロセスに組み込む
- 📝 **注意**: UI を Figma に送信するには、リモート MCP サーバーが必要です

## GitHub Copilot in Visual Studio Code v1.110 – February release

**URL:** https://github.blog/changelog/2026-03-06-github-copilot-in-visual-studio-code-v1-110-february-release

### 要約
VS Code の 2026 年 2 月リリースでは、エージェントがより長時間かつ複雑なタスクに対応できるようになりました。ライフサイクルフックによる自動化、会話の分岐、チャットからの自動承認、エージェントプラグイン、ブラウザツール、スキルのスラッシュコマンド化など、エージェントの制御性と拡張性が大幅に向上しています。さらに、Copilot CLI が VS Code に統合され、エージェント間でのメモリ共有やコンテキスト圧縮などのスマートなセッション管理機能も追加されました。

### ユーザーにとって良いこと
- **エージェントの制御性向上**: フック、分岐、キューイングにより、エージェントの動作をより細かく制御でき、長時間タスクでも安心して任せられます
- **拡張性の強化**: エージェントプラグイン、スキル、カスタマイズをチャットから直接作成できるため、自分のワークフローに合わせてエージェントを拡張可能
- **コンテキスト管理の改善**: メモリ共有、プラン永続化、自動コンテキスト圧縮により、エージェントが過去の作業を記憶し続け、毎回ゼロから始める必要がなくなります
- **生産性の向上**: Copilot CLI の統合、ターミナルでの画像表示、モデルピッカーの改善など、日常のコーディング体験が全体的に向上します

### アクションプラン
- **VS Code を最新版に更新**: v1.110 以降にアップデートして新機能を有効化
- **エージェントのフックを試す**: ライフサイクルイベントでコードを実行し、自動リント、ポリシー適用、コマンド実行前のブロックなどを試す
- **会話の分岐を活用**: `/fork` で会話を分岐し、異なるアプローチを探索する
- **自動承認を設定**: `/autoApprove` または `/yolo` でグローバル自動承認を有効化し、ターミナルサンドボックスと組み合わせて安全に使用
- **エージェントプラグインをインストール**: Extensions ビューからプラグインを検索し、スキル・ツール・フック・MCP サーバーのバンドルをインストール（実験的機能として `/experimental` で有効化）
- **スラッシュコマンドでスキルを呼び出す**: エージェントスキルを `/` コマンドとしてチャットから直接トリガー
- **ブラウザツールを試す**: 統合ブラウザを使って、エージェントにナビゲート、クリック、スクリーンショット、変更検証をさせる（実験的機能）
- **カスタマイズをチャットから作成**: `/create-*` コマンドで、プロンプト、スキル、エージェント、フックを会話から直接生成
- **Copilot CLI を活用**: VS Code に統合された Copilot CLI を使い、diff タブ、信頼フォルダー同期、右クリックでのコード送信などを試す
- **コンテキスト圧縮を手動実行**: `/compact` でコンテキストを圧縮し、必要な情報を保持（例: `/compact forget about all variants, except the rust version`）
- **Explore サブエージェントを活用**: コードベースの調査を高速化し、Plan エージェントが具体的なファイルやコードパスを参照できるようにする
- **AI コミット属性を確認**: AI が生成したコードを含むコミットに自動的に属性が付与されるため、履歴管理に役立てる
