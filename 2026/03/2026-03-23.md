## Gemini 3.1 Pro is now available in JetBrains IDEs, Xcode, and Eclipse

**URL:** https://github.blog/changelog/2026-03-23-gemini-3-1-pro-is-now-available-in-jetbrains-ides-xcode-and-eclipse

### 要約
Gemini 3.1 Pro が Copilot Enterprise, Business, Pro, そして Pro+ 向けに public preview で利用可能になりました。GitHub.com、GitHub Mobile、Visual Studio Code、Visual Studio、JetBrains IDEs、Xcode、Eclipse の Copilot Chat から、agent / ask / edit モードでこのモデルを選べます。

### ユーザーにとって良いこと
普段使っている IDE や GitHub のチャット画面から、より多くの環境で同じモデルを利用できるようになります。チームはツールを切り替えずに試せるので、モデル評価や導入判断がしやすくなります。

### アクションプラン
- Copilot Business / Enterprise 管理者は、Copilot settings で Gemini 3.1 Pro policy を有効化してください。
- 対象ユーザーに、利用可能になるまで表示されない場合があることを周知してください。
- 既存の Copilot モデル評価フローがある場合は、JetBrains / Xcode / Eclipse を含めて再確認してください。
- ⚠️ public preview なので、本番運用で全面採用する前に、少人数で検証してください。

## Push protection exemptions from repository settings

**URL:** https://github.blog/changelog/2026-03-23-push-protection-exemptions-from-repository-settings

### 要約
Secret scanning の push protection exemption を、organization や enterprise だけでなく repository settings からも設定できるようになりました。これにより、特定の role、team、app を push protection の対象外として指定できます。

### ユーザーにとって良いこと
リポジトリ単位で例外を管理できるため、より細かい運用がしやすくなります。push ごとに exemption が評価されるので、条件に合う actor には bypass request を発生させず、運用負荷を下げられます。

### アクションプラン
- 既存の secret scanning / push protection ポリシーを確認し、repo 単位の例外設定が必要か判断してください。
- 例外対象の role / team / app を棚卸しし、最小権限になるよう見直してください。
- ⚠️ 例外を広く設定すると secrets の流出リスクが上がるため、設定範囲を最小限にしてください。
- 運用ルールとして、repo settings と org / enterprise settings のどちらで管理するかを明確にしてください。
