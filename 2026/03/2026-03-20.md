## Copilot usage metrics now resolve auto model selection to actual models

**URL:** https://github.blog/changelog/2026-03-20-copilot-usage-metrics-now-resolve-auto-model-selection-to-actual-models

### 要約
Copilotの使用状況メトリクスで「Auto」選択時も実際のモデル名が表示されるようになりました。これにより、管理者はどのモデルが使われているかを正確に把握できます。REST APIやダッシュボードのモデル別集計に反映されます。

### ユーザーにとって良いこと
組織内でどのAIモデルが利用されているかを明確に把握でき、コンプライアンスや監査対応が容易になります。

### アクションプラン
- Copilotの使用状況レポートやAPIを利用している場合、モデル別集計の内容を確認・活用してください。
- 自動選択（Auto）モードの利用が多い場合も、モデルごとの利用状況を把握できます。

### リファレンス
- [Copilot usage metrics now resolve auto model selection to actual models](https://github.blog/changelog/2026-03-20-copilot-usage-metrics-now-resolve-auto-model-selection-to-actual-models)

## Trace any Copilot coding agent commit to its session logs

**URL:** https://github.blog/changelog/2026-03-20-trace-any-copilot-coding-agent-commit-to-its-session-logs

### 要約
Copilot coding agentが作成したコミットに、セッションログへのリンク（Agent-Logs-Url）が付与されるようになりました。これにより、なぜその変更が行われたかを後から追跡できます。

### ユーザーにとって良いこと
エージェントが生成したコードの意図や経緯を簡単に確認でき、監査やレビューがしやすくなります。

### アクションプラン
- Copilot coding agentのコミットを確認する際は、コミットメッセージ内のAgent-Logs-Urlを参照してください。
- 監査やレビュー時にセッションログを活用し、変更理由を把握しましょう。

### リファレンス
- [Trace any Copilot coding agent commit to its session logs](https://github.blog/changelog/2026-03-20-trace-any-copilot-coding-agent-commit-to-its-session-logs)

## Monitor Copilot coding agent logs live in Raycast

**URL:** https://github.blog/changelog/2026-03-20-monitor-copilot-coding-agent-logs-live-in-raycast

### 要約
Raycast用GitHub Copilot拡張で、Copilot coding agentのログをリアルタイムで確認できるようになりました。Raycastから直接進捗を追跡できます。

### ユーザーにとって良いこと
GitHubにアクセスせずに、Raycast上でエージェントの進捗やログを素早く確認できます。

### アクションプラン
- Raycast拡張をインストールし、「View Tasks」コマンドからセッションを選択してログを確認してください。
- Copilot coding agentの利用には管理者による有効化が必要な場合があります。

### リファレンス
- [Monitor Copilot coding agent logs live in Raycast](https://github.blog/changelog/2026-03-20-monitor-copilot-coding-agent-logs-live-in-raycast)
