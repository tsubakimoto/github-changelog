## Copilot usage metrics now resolve auto model selection to actual models

**URL:** https://github.blog/changelog/2026-03-20-copilot-usage-metrics-now-resolve-auto-model-selection-to-actual-models

### 要約
Copilot usage metrics で auto model selection を利用している場合でも、従来の汎用的な `Auto` 表示ではなく、実際に使われたモデル名で利用状況を確認できるようになりました。REST API とダッシュボードの両方で、`enterprise`、`org`、`user` レベルの `totals_by_model_feature` や `Model usage per chat mode` の集計が、実モデルベースで見えるようになります。

### ユーザーにとって良いこと
組織内でどのモデルが実際に使われているかを正確に把握できるため、監査、コンプライアンス、コスト分析がしやすくなります。auto モードの採用が増えても、モデルごとの利用傾向を見失いにくくなります。

### アクションプラン
- Copilot 管理者は usage metrics ダッシュボードや REST API のモデル別集計を見直し、これまで `Auto` として扱っていた社内レポートや可視化ロジックを更新
- `totals_by_model_feature` を使った BI / 監査レポートがある場合は、実モデル名に分解される前提でクエリと集計単位を再確認
- 自動適用の改善なので設定変更は不要。ただし auto 利用と手動選択の内訳はまだ分かれないため、必要なら補助指標を別途設計

## Trace any Copilot coding agent commit to its session logs

**URL:** https://github.blog/changelog/2026-03-20-trace-any-copilot-coding-agent-commit-to-its-session-logs

### 要約
Copilot coding agent が作成した commit には、Copilot が author、依頼した人が co-author として記録されるだけでなく、新たに `Agent-Logs-Url` trailer が付くようになりました。これにより、各 commit から対応する agent session logs へ直接たどれ、変更の背景や実行経緯をあとから追跡しやすくなります。

### ユーザーにとって良いこと
コードレビュー時に「なぜこの変更が入ったのか」を commit からすぐ確認でき、監査や事後調査の透明性が上がります。agent が関与した変更と、その作業を開始した人の関係も追いやすくなります。

### アクションプラン
- Copilot coding agent を使うチームでは、reviewer 向け手順に「agent authored commit の `Agent-Logs-Url` を確認する」流れを追加
- commit message trailer を解析する社内ツールや監査基盤がある場合は、`Agent-Logs-Url` 行を許容または活用できるか確認
- Copilot Business / Enterprise で未導入なら、管理者は `Policies` 画面で Copilot coding agent の有効化状況を確認

## Monitor Copilot coding agent logs live in Raycast

**URL:** https://github.blog/changelog/2026-03-20-monitor-copilot-coding-agent-logs-live-in-raycast

### 要約
GitHub Copilot extension for Raycast から、Copilot coding agent の live logs を直接見られるようになりました。Raycast の `View Tasks` コマンドでセッションを選ぶだけで、GitHub を開かずに agent の進捗を追跡できます。

### ユーザーにとって良いこと
別の作業を続けながら、少ない操作で Copilot の進み具合を確認できます。ブラウザと行き来する手間が減るため、agent への委譲を日常的に使いやすくなります。

### アクションプラン
- Raycast を使っている開発者は GitHub Copilot extension をインストールまたは更新し、`View Tasks` から live logs を確認できるか試す
- Copilot coding agent を使う運用がある場合は、ブラウザ監視だけでなく Raycast 監視も選択肢として社内ガイドに追記
- オプトイン機能なので Raycast 利用者だけが対象。Copilot Business / Enterprise では管理者が coding agent を有効化していることも確認

### リファレンス
- [Monitor Copilot coding agent logs live in Raycast](https://github.blog/changelog/2026-03-20-monitor-copilot-coding-agent-logs-live-in-raycast)

## A smoother navigation experience in GitHub Mobile for Android

**URL:** https://github.blog/changelog/2026-03-20-a-smoother-navigation-experience-in-github-mobile-for-android

### 要約
GitHub Mobile for Android のナビゲーション体験が見直され、`Home` や `Inbox` など主要エリア間の移動がより自然になりました。bottom navigation が必要な場面でより一貫して表示され、各タブでも直前の位置が保たれやすくなっています。

### ユーザーにとって良いこと
コンテキストを切り替えても今見ていた場所に戻りやすく、通知確認や issue / PR の往復がスムーズになります。モバイルで GitHub を使う頻度が高い人ほど、操作の迷いと移動コストを減らせます。

### アクションプラン
- Android 利用者は GitHub Mobile を最新の production build へ更新し、新しい navigation を確認
- 社内 onboarding 資料やモバイル操作手順に旧 UI のスクリーンショットがある場合は、画面遷移の説明を更新
- アプリ更新で利用できる改善なので追加設定は不要。チームで Android 利用者が多ければ軽い周知を実施

### リファレンス
- [A smoother navigation experience in GitHub Mobile for Android](https://github.blog/changelog/2026-03-20-a-smoother-navigation-experience-in-github-mobile-for-android)
