## Ask @copilot to make changes to any pull request

**URL:** https://github.blog/changelog/2026-03-24-ask-copilot-to-make-changes-to-any-pull-request

### 要約
Pull request 内で `@copilot` をメンションすると、Copilot coding agent に修正作業を依頼できるようになりました。失敗した GitHub Actions の修正、レビューコメントへの対応、テスト追加などを、既存の PR をベースにしたまま進められます。

### ユーザーにとって良いこと
レビューや CI 修正のたびに新しい PR を切り直す必要が減り、既存の会話や差分の流れを保ったまま作業できます。Copilot が自分の開発環境で変更・検証・push まで行うので、修正の往復コストも下がります。

### アクションプラン
- PR 上で `@copilot` を使う運用をチームに共有し、レビューコメント対応や CI 修正の標準フローに組み込んでください。
- Copilot Business / Enterprise を使っている場合は、管理者が Copilot coding agent を有効化しているか確認してください。
- 既存の「修正用に新しい PR を作る」手順が残っている場合は、`@copilot` への置き換え可否を見直してください。
- ⚠️ coding agent は有料 Copilot プラン向けで、Business / Enterprise では管理者の有効化が必要です。

## Manage Copilot coding agent repository access via the API

**URL:** https://github.blog/changelog/2026-03-24-manage-copilot-coding-agent-repository-access-via-the-api

### 要約
Copilot coding agent のリポジトリアクセスを REST API から管理できるようになりました。組織ごとに、利用可能なリポジトリを「なし」「すべて」「選択したリポジトリ」から制御でき、選択方式のときは許可リポジトリの追加・削除もできます。

### ユーザーにとって良いこと
大規模な組織でも、UI を手作業で触らずにアクセス設定を自動化できます。既存の権限管理やプロビジョニング処理に組み込めるため、設定のばらつきや運用ミスを減らしやすくなります。

### アクションプラン
- organization owners は、既存の権限管理スクリプトに Copilot coding agent の API 操作を組み込めるか確認してください。
- 「選択したリポジトリのみ」を使う場合は、許可対象リポジトリの棚卸しを行い、追加・削除の手順を定義してください。
- 管理画面での設定変更が残っているなら、API ベースの運用へ寄せて一元化してください。
- ⚠️ public preview のため、本番の自動化に入れる前に検証環境で挙動を確認してください。

## Faster incremental analysis with CodeQL in pull requests

**URL:** https://github.blog/changelog/2026-03-24-faster-incremental-analysis-with-codeql-in-pull-requests

### 要約
CodeQL の pull request スキャンが、C#、Java、JavaScript/TypeScript、Python、Ruby でさらに高速化されました。新規・変更コードを表す database と、既存のキャッシュ済み database を組み合わせることで、default CodeQL query suite の incremental analysis がより効率的になります。

### ユーザーにとって良いこと
PR 上のセキュリティ分析が短時間で終わりやすくなり、待ち時間の削減につながります。大きめのリポジトリほど恩恵が出やすく、レビューの流れを止めにくくなります。

### アクションプラン
- CodeQL を使っているリポジトリで、対象言語が C# / Java / JavaScript(TypeScript) / Python / Ruby か確認してください。
- build mode `none` の既定セットアップまたは advanced setup を使っている場合は、incremental analysis が有効になっているか確認してください。
- default CodeQL query suite 以外を使っている場合は、今回の高速化が適用されない点を周知してください。
- ⚠️ CodeQL CLI の incremental scanning 対応は後日提供予定のため、CLI 前提の運用では別途ロードマップを確認してください。
