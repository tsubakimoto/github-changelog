## Configure Copilot coding agent’s validation tools

**URL:** https://github.blog/changelog/2026-03-18-configure-copilot-coding-agents-validation-tools

### 要約
Copilot coding agent が実行する validation tools を、repository admins が設定できるようになりました。テストや linter に加え、GitHub 側の security / quality validation tools のうち、必要なものだけを選んで実行対象にできます。

### ユーザーにとって良いこと
プロジェクトに合わない長時間チェックや不要な検証を減らしつつ、必要な品質ゲートは維持できます。Copilot の自動修正フローを、チームの開発規約や CI 特性に合わせやすくなります。

### アクションプラン
- repository admins は **Copilot → Coding agent** 設定を確認し、実行したい validation tools を棚卸し
- 長時間実行やノイズになっている validation があれば、影響を見ながら無効化を検討
- 明示設定が必要な管理機能なので、リポジトリごとの差分を運用ドキュメントへ反映

## GPT-5.3-Codex long-term support in GitHub Copilot

**URL:** https://github.blog/changelog/2026-03-18-gpt-5-3-codex-long-term-support-in-github-copilot

### 要約
GitHub Copilot に GPT-5.3-Codex の long-term support (LTS) が導入されました。Copilot Business / Enterprise 向けに 12 か月の提供期間が保証され、さらに GPT-5.3-Codex は新しい base model として May 17, 2026 までに順次有効化されます。

### ユーザーにとって良いこと
エンタープライズでは、モデルの安定性や内部レビューの計画を立てやすくなります。既知のモデル挙動を長く使えるため、回帰検証やコンプライアンス対応に向いています。

### アクションプラン
- Copilot Business / Enterprise 管理者は、GPT-5.3-Codex の利用可否と内部承認状況を確認
- ⚠️ `2026-05-17` までに base model へ切り替わる前提で、既存ワークフローの検証を進める
- ⚠️ LTS 提供終了の `2027-02-04` を見据え、中長期のモデル更新計画を用意
- premium request unit multiplier も含め、コストと運用ポリシーを関係者へ共有
