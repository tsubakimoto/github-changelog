## GitHub Copilot for Jira — Public preview enhancements

**URL:** https://github.blog/changelog/2026-03-25-github-copilot-for-jira-public-preview-enhancements

### 要約
GitHub Copilot for Jira の公開プレビューに、オンボーディング改善、モデル選択、Jira チケット番号の PR 連携、Confluence コンテキストの MCP 連携が追加されました。初期設定時のエラー説明や前提条件の案内が強化され、Jira から Copilot coding agent をより細かく制御できるようになっています。

### ユーザーにとって良いこと
Jira からそのまま Copilot を使う際の設定ミスを減らし、作業の意図や参照情報を PR に自動で残しやすくなります。Confluence の設計資料も参照できるため、チケット駆動の実装フローがより一貫しやすくなります。

### アクションプラン
- Jira 連携を使っているチームは、最新の GitHub Copilot for Jira アプリに更新してください。
- 使いたい AI モデルがある場合は、Jira コメント内の `@GitHub Copilot` 呼び出しでモデル指定の運用を確認してください。
- ⚠️ Jira チケット番号の PR タイトル・ブランチ命名ルールが既存の運用と競合しないか確認してください。
- Confluence を参照したい場合は、Atlassian MCP サーバーと PAT の設定を追加してください。

## Updates to our Privacy Statement and Terms of Service: How we use your data

**URL:** https://github.blog/changelog/2026-03-25-updates-to-our-privacy-statement-and-terms-of-service-how-we-use-your-data

### 要約
GitHub は Privacy Statement と Terms of Service を更新し、Copilot Free / Pro / Pro+ の入力・出力・コードスニペット・関連コンテキストを AI モデルの改善に利用する方針を明確化しました。企業向けの Copilot Business / Enterprise は対象外で、個人向けユーザーは 4 月 24 日以降にオプトアウト可能です。

### ユーザーにとって良いこと
何が収集・利用されるか、どの製品が対象か、オプトアウト方法が明確になりました。個人利用者は設定で制御でき、企業アカウントは従来どおり保護されるため、利用条件を判断しやすくなります。

### アクションプラン
- Copilot Free / Pro / Pro+ を使う個人アカウントは、4 月 24 日までにプライバシー設定を確認してください。
- 収集を望まない場合は、設定の `Privacy` からオプトアウトしてください。
- 企業・組織アカウントの管理者は、今回の変更が対象外であることを社内ポリシーに反映してください。
- ⚠️ チーム内で個人アカウントを業務利用している場合は、学習対象になる可能性を周知してください。

## Copilot usage metrics now identify active Copilot coding agent users

**URL:** https://github.blog/changelog/2026-03-25-copilot-usage-metrics-now-identify-active-copilot-coding-agent-users

### 要約
Copilot の利用メトリクスで、Copilot coding agent を使っているユーザーを識別できるようになりました。API では `used_copilot_coding_agent` フィールドが追加され、ダッシュボードや日次・28 日レポートで IDE 利用とエージェント利用を区別できます。

### ユーザーにとって良いこと
Copilot の導入状況を IDE だけでなく、自律実行の coding agent まで含めて把握できます。利用実態に応じた教育や導入支援、予算判断がしやすくなります。

### アクションプラン
- Enterprise / Organization 管理者は、既存の Copilot 利用レポートを `used_agent` と `used_copilot_coding_agent` の両方で見直してください。
- API を利用している場合は、新しい `used_copilot_coding_agent` フィールドを取り込むよう集計処理を更新してください。
- Copilot の定着状況を測る指標に、issue 割り当てや PR コメント経由の利用を追加してください。

## Disable comments on individual commits

**URL:** https://github.blog/changelog/2026-03-25-disable-comments-on-individual-commits

### 要約
リポジトリ管理者が、個別コミットへのコメントを無効化できるようになりました。設定の `Allow comments on individual commits` をオフにすると、コメント UI と API 経由の新規コメント作成が止まり、既存コメントは閲覧・編集・削除のみ可能です。

### ユーザーにとって良いこと
古いコミットへの不要なコメントやノイズを抑えやすくなり、メンテナンス負荷を減らせます。既存コメントは保持されるため、履歴を壊さずに運用方針を変えられます。

### アクションプラン
- ノイズが多いリポジトリの管理者は、Settings の `Commits` セクションで設定を確認してください。
- コメントを止めたい場合は、`Allow comments on individual commits` をオフにしてください。
- ⚠️ REST API と GraphQL API からの新規コメントもブロックされるため、自動化や外部ツールの影響を確認してください。
