## More visibility into Copilot coding agent sessions

**URL:** https://github.blog/changelog/2026-03-19-more-visibility-into-copilot-coding-agent-sessions

### 要約
Copilot coding agentのセッションログが大幅に改善され、セットアップ手順やサブエージェントの動作がより明確に可視化されるようになりました。カスタムセットアップ手順の出力もログに表示され、デバッグが容易になりました。

### ユーザーにとって良いこと
Copilotの動作状況やセットアップの進行状況が分かりやすくなり、問題発生時の原因特定や環境構築の確認がしやすくなります。

### アクションプラン
- セッションログを活用してCopilotの動作やセットアップ手順を確認する
- `copilot-setup-steps.yml`を利用している場合は、出力内容をチェックし、必要に応じて修正
- サブエージェントの動作もログで確認し、詳細が必要な場合は展開して内容を把握

### リファレンス
- [More visibility into Copilot coding agent sessions](https://github.blog/changelog/2026-03-19-more-visibility-into-copilot-coding-agent-sessions)

## GitHub Actions: Late March 2026 updates

**URL:** https://github.blog/changelog/2026-03-19-github-actions-late-march-2026-updates

### 要約
GitHub Actionsで、環境を自動デプロイせずに利用できる機能や、スケジュールワークフローでタイムゾーン指定が可能になりました。細かな使い勝手の向上が図られています。

### ユーザーにとって良いこと
環境のシークレットや変数管理だけを使いたい場合に余計なデプロイが不要になり、cronスケジュールもローカルタイムで設定できるため運用が柔軟になります。

### アクションプラン
- `deployment: false`を設定し、必要に応じて環境のみを利用
- スケジュールワークフローで`timezone`フィールドを追加し、希望のタイムゾーンで実行
- カスタムデプロイ保護ルール利用時は自動デプロイが必須な点に注意

### リファレンス
- [GitHub Actions: Late March 2026 updates](https://github.blog/changelog/2026-03-19-github-actions-late-march-2026-updates)
