## GPT-5.4 is generally available in GitHub Copilot

**URL:** https://github.blog/changelog/2026-03-05-gpt-5-4-is-generally-available-in-github-copilot

### 要約
OpenAIの最新agentic codingモデルであるGPT-5.4がGitHub Copilotで一般利用可能になりました。複雑なマルチステップタスクで強化された論理推論とタスク実行能力を提供します。Copilot Pro、Pro+、Business、Enterpriseユーザーが利用でき、Visual Studio Code、Visual Studio、JetBrains、Xcode、Eclipse、GitHub.comなどで選択可能です。

### ユーザーにとって良いこと
より高度なコード生成と複雑なロジック構築のサポートが得られます。複数ステップの開発タスクをより正確に自動化でき、開発効率が向上します。

### アクションプラン
- Copilot Pro/Pro+/Business/Enterpriseユーザーは最新バージョン（VS Code v1.104.1以上、Visual Studio 17.14.19以上など）にアップグレード
- Copilot Enterprise/Businessプランの管理者はCopilot設定でGPT-5.4ポリシーを有効化
- 利用可能なモデル一覧を確認し、必要に応じてプロンプトを調整

### リファレンス
- [GPT-5.4 is generally available in GitHub Copilot](https://github.blog/changelog/2026-03-05-gpt-5-4-is-generally-available-in-github-copilot)

## Hierarchy view improvements and file uploads in issue forms

**URL:** https://github.blog/changelog/2026-03-05-hierarchy-view-improvements-and-file-uploads-in-issue-forms

### 要約
GitHub Projectsのhierarchy viewに3つの改善が加わりました。サブイシュー（sub-issues.is:openなど）のフィルタ、重複イシューの除去、展開/折りたたみ状態の保存です。また、イシューフォームで新しいアップロードフィールドタイプが追加され、ログやスクリーンショットを必須化できます。さらに、イテレーションビューで複数アイテムを一括移動可能になりました。

### ユーザーにとって良いこと
プロジェクト管理がより効率的になり、イシュー収集時に必要なファイルを事前に取得でき、スプリント計画の管理が簡素化されます。

### アクションプラン
- GitHub Projectsで新しいフィルタ構文（sub-issues.is:openなど）を確認・活用
- イシューフォームの設定でアップロードフィールドを追加（ログやスクリーンショット必須化）
- イテレーション管理での一括移動機能を試用（グループヘッダーメニューから実行）

### リファレンス

## GitHub Copilot coding agent for Jira is now in public preview

**URL:** https://github.blog/changelog/2026-03-05-github-copilot-coding-agent-for-jira-is-now-in-public-preview

### 要約
GitHub Copilot coding agentがJira統合を提供し、Jiraイシューに割り当てると自動的にGitHub上でドラフトプルリクエストを生成します。エージェントはJiraイシューを分析し、独立して変更を実装し、Jira内で更新をポストできます。

### ユーザーにとって良いこと
Jira-GitHub間のコンテキストスイッチが削減され、バグ修正やドキュメント更新などの反復的タスクが加速します。既存のレビュー・承認ルールが適用されます。

### アクションプラン
- Atlassian MarketplaceからGitHub Copilot for Jiraアプリをインストール
- セットアップ中にGitHubアプリをインストール
- GitHub Organization接続時にCopilotがアクセス可能なリポジトリを設定
- Jiraイシューに対してCopilot coding agentを割り当てて試用開始
- 要件確認：Jira CloudでRovo有効、Copilot coding agent有効、接続済みGitHubリポジトリ必須

### リファレンス
- [GitHub Copilot coding agent for Jira is now in public preview](https://github.blog/changelog/2026-03-05-github-copilot-coding-agent-for-jira-is-now-in-public-preview)

## Copilot code review now runs on an agentic architecture

**URL:** https://github.blog/changelog/2026-03-05-copilot-code-review-now-runs-on-an-agentic-architecture

### 要約
Copilot code reviewがagentic tool-calling architectureで一般利用可能になり、より広範なリポジトリコンテキスト（関連コード、ディレクトリ構造、参照など）を収集してレビュー品質を向上させました。より高品質で実行可能なレビューコメントが提供されるようになります。

### アクションプラン
- Copilot Pro/Pro+/Business/Enterpriseユーザーはドキュメント確認してコードレビュー機能を有効化
- ⚠️ セルフホストランナーを使用している場合：agentic code reviewの実行にはワンタイム設定が必須（GitHub Actions実行環境の設定必要）
- プルリクエストでレビューコメント品質を検証・フィードバック提供
